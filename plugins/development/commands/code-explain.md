## Task
Read and explain the code in the specified file. Provide:

1. **Overview**: High-level purpose and functionality
2. **Key Components**: Main functions, classes, or modules
3. **Logic Flow**: How the code works step-by-step
4. **Dependencies**: Important imports and external dependencies
5. **Notable Patterns**: Design patterns or architectural decisions

Keep the explanation clear and concise, focusing on what the code does and why.

### Arguments
If a file path is provided as an argument, explain that file. Otherwise, ask the user which file to explain.
